## Pitfalls Documented

- Memory fragmentation: at 90%+ usage, distillation drops granular details. Agent "remembers" a repo exists but forgets branch/name/path, causing duplicate/wrong actions.
- Skill drafts in `managed-agents/internal/skills-queue/` are historical bureaucracy. Correct flow: discovery → direct `skill_manage(action="create")` → done.
- GitHub raw curl bypass works for public repos but requires `jq`/`sed` fluency and cannot substitute actual runtime verification.
